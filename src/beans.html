Fresh News & Blogs API

Beans is our aggregation and search service for news and blogs. It supports semantic query search, practical filtering, and clean JSON output. Think of it as your content firehose with better manners.

Core Endpoints

GET /health

Simple liveness check. Returns status alive for load balancers and uptime checks.

GET /tags/categories

Returns category tags with pagination using offset and limit (default 0 and 16).

GET /tags/entities

Named entities discovered in indexed content. Useful for faceted filtering and UI chips.

GET /tags/regions

Region tags for geography-aware filtering and analytics dashboards.

GET /publishers

Publisher metadata by source list. Returns source, site name, base URL, and favicon.

GET /publishers/sources

Unique source identifiers currently represented in the database.

GET /articles/latest

Main search route. Supports semantic query q, threshold acc, filters, and paging.

GET /articles/trending

Same filters as latest but sorted by trend score for high-momentum stories.

Filter + Search Example

Yes, the route names are boring. That is intentional. Boring route names fail less in production. 

BASE_URL="http://localhost:8080"
API_KEY="my-secret"

curl -s \
  -H "Content-Type: application/json" \
  -H "X-API-KEY: $API_KEY" \
  "$BASE_URL/articles/latest?q=artificial+intelligence&acc=0.8&kind=news&categories=AI,Tech&offset=0&limit=5"
  • q: semantic query text (3-512 chars)
  • acc: cosine similarity threshold from 0 to 1
  • kind: news or blog
  • categories, entities, regions, sources: comma-separated filters

Response Shape

Primary routes return arrays of Bean or BeanAggregate objects.

{
  "url": "https://example.com/story",
  "kind": "news",
  "source": "example.com",
  "title": "Breaking AI News",
  "summary": "A short summary...",
  "author": "Jane Doe",
  "created": "2024-02-28T12:34:56Z",
  "entities": ["OpenAI", "GPT"],
  "regions": ["US"],
  "categories": ["AI"]
}
  • JSON everywhere
  • Auth optional unless API_KEYS is set
  • HTTP status codes reflect success or failure

Ops and Deployment Notes

Build and Run

  • Build with go build
  • Run binary directly or via docker-compose
  • Use .env for local environment config

Important Environment Variables

  • PORT (default 8080)
  • PG_CONNECTION_STRING for PostgreSQL
  • EMBEDDER_BASE_URL and EMBEDDER_API_KEY
  • MAX_CONCURRENT_REQS for throughput control

MCP Positioning
